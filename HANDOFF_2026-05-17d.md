## What landed this session

### SCT-9 partial — two commits in corpus

**Commit 1: `664d633`** — `POS(0)/RPOS(0)` sentinels in all six `parser_*.sc` Compilands

Root cause of empty-tree output diagnosed and fixed. SPITBOL's `ARBNO` is
lazy/shy: it matches zero iterations when a trailing epsilon branch allows
the overall match to succeed. All six parsers had:

```
Compiland = nPush() ARBNO(...) reduce(..., 'nTop()') (END...|epsilon) nPop()
```

The epsilon made ARBNO stop at 0 iterations, so `nTop()=0`, and every parse
produced an empty tree ("Parse Error" or zero children).

Fix: added `POS(0)` before `ARBNO` (anchor to start of subject) and `RPOS(0)`
after (assert EOS, forcing ARBNO to consume all input before `reduce` fires).
SCRIP's native ARBNO is greedy; this makes SPITBOL match it.

Results after fix (using transpile chain with `gen.sc` included):
- snobol4 ✅ — `\tX = 1\nEND\n` → `(TT_STMT () (TT_LABEL) (TT_EQ (TT_VAR X) (TT_ILIT 1)))`
- snocone ✅ — `x = 1+2;` → `(STMT :eq :subj (TT_VAR x) :repl (TT_ADD ...))`
- rebus ✅ — 88/96 corpus fixtures produce correct AST
- icon — parse blocked on `notmatch`/`Push_qlit` from `icon_helpers.sc` (SCT-4 prereq)
- raku — ERROR 022, missing helper functions (SCT-5 prereq)
- prolog — ERROR 022, missing helper functions (SCT-6 prereq)

## Critical implementation notes for next session

### Transpile chain requires `gen.sc`

The `--dump-sno` transpile chain must include `gen.sc` alongside `tdump.sc`:

```bash
./scrip --dump-sno \
  $SC/global.sc $SC/tree.sc $SC/stack.sc $SC/counter.sc \
  $SC/ShiftReduce.sc $SC/semantic.sc $SC/gen.sc $SC/tdump.sc \
  $SC/parser_<lang>.sc > /tmp/parser_<lang>.sno
```

`gen.sc` provides `Gen`, `GetLevel`, `IncLevel`, `DecLevel` — called by
`TDump` in `tdump.sc`. Without it, all parsers run but produce zero output
(TDump silently calls undefined functions which fail quietly).

This is NOT yet reflected in `scripts/run_parser_sync_monitor.sh` — that
script needs updating to add `gen.sc` to its invocation.

### SPITBOL's ARBNO is lazy — future parser work

Any future `parser_*.sc` Compiland-level loop that uses `ARBNO` with
side-effecting contents (`nInc`, `nPush`, `shift`, `reduce`, etc.) MUST
have `POS(0) ARBNO(...) RPOS(0)` to work correctly under SPITBOL. SCRIP's
native runtime is greedy; SPITBOL's is not.

Inner ARBNOs (e.g. `ARBNO(BREAK(nl) nl)`, `ARBNO(white)`) do NOT need
sentinels — they have no side effects and appear inside sub-patterns, not
at Compiland top level.

### TDump formatting gap

The transpiled TDump emits compact inline output:
```
(STMT :eq :subj (TT_VAR x) :repl (TT_ADD (TT_ILIT 1) (TT_ILIT 2)))
```

The `.ref` files use indented multi-line format:
```
(STMT :eq :subj (TT_VAR x) :repl (TT_ADD
  (TT_ILIT 1)
  (TT_ILIT 2)
))
```

The gate comparison (`test_parser_sc_transpile.sh`, once written) must
normalise whitespace, or the transpiled TDump must be made to produce
indented output matching the refs. The whitespace normalisation approach
is simpler: strip all whitespace from both sides before comparing.
