## FIX 1 (LANDED): Icon set operators `++` `--` `**` were broken

### Symptom
`hello, world` threw a JVM NullPointerException (`<local18> is null`). `javap -c` diff vs the
canonical-jtran `.class` showed SCRIP-jtran dropped the undeclared-variable bytecode preamble
(`getstatic v$write$ / dup / ifnonnull / pop / ldc "write" / NewLocal / astore`). Root: in
`gen_bc.icn`, `bc_createvars`' `undeclared` set (`bc_var_set -- bc_params_set -- bc_locals_set --
bc_statics_set`, `--` = SET DIFFERENCE) came out empty under SCRIP.

Minimized: `set ++ set` → 10 (oracle 3), `set ** set` → 10 (oracle 1), `set -- set` → 0 (oracle 2).
cset `--` was correct (last session's CDIFF fix). SO: SCRIP routed SET operands through the CSET
path.

### Root cause
`BINOP_CUNION/CDIFF/CINTER` are OVERLOADED in Icon (canonical `oset.r`: `if is:set(x) &&
is:set(y) then <set op> else <cset op>`). SCRIP only implemented the cset arm; a `DT_T`-with-
`is_set` operand fell through to `VARVAL_fn` (string/cset path). Both dispatch sites had the bug:
`src/runtime/arithmetic.c` (compiled/m4 path) and `src/runtime/by_name_dispatch.c` (by-name).

### Fix
- `src/runtime/aggregates.c`: added `set_union` / `set_diff` / `set_inter` (iterate x's buckets,
  test membership in y via `table_has`, copy `key`/`key_descr` into a fresh `is_set` TBBLK).
- `src/runtime/core/core.h`: declared the three.
- `arithmetic.c` + `by_name_dispatch.c`: added `both-operands-are-set` guard at the top of the
  CUNION/CDIFF/CINTER branch → dispatch to the set ops, returning `TABLE_VAL(...)`.

### Verified
set ++/--/** byte-identical to oracle (m3 and m4), cset `--` still correct, elements (not just
sizes) match, chained `--` matches. Icon smoke 14/14 both modes. This is what makes `hello, world`
work end-to-end.

## FIX 2 (LANDED): `while`/`until` with an ALTERNATION condition looped once / diverged

### Symptom
`while x = 1 | x = 2 do {...}` executed the body ONCE (oracle: until condition truly fails).
`until x=1|x=2 do` diverged (11 iters vs 3). Single-condition while/until fine; `if`-alt fine.

### Root cause
`lower_while` / `lower_until` (`src/lower/lower_icon.c`) wired the body's loop-back edge DIRECTLY
to the condition entry (`lower(cx, B, centry, centry, ...)`). When the condition is generator-kind
(a DISJUNCTION, i.e. an alternation), the body's terminal edge β-PROMOTES, so on the next lap the
condition is RESUMED (advance to next alt / exhaust) instead of RESTARTED from α. Icon `while`/
`until` re-evaluate the condition FRESH each lap.

### Fix
Added an α-force trampoline `CENT = build(IR_GOTO)` with `lc_γ_to_α(CENT, centry)` /
`lc_ω_to_α(CENT, centry)`, set `cx->loop_next = CENT`, and lowered the body targeting `CENT`
instead of `centry`. Mirrors the existing `lower_every` mark→body α-force (same file, ~line 1104)
and `lower_until`'s BENT idiom.

### Verified
`while x=1|x=2` → 3 iters; `until` alt → 3 iters / correct final. Icon rung suite **241/20/32**
(byte-identical FAIL set to the pre-change baseline in the LIVE CURSOR — ZERO regression). Smoke
14/14 ×2. Gates no_stack / one_reg / semicolon all green.

## BLOCKER (DIAGNOSED, NOT FIXED): `===` (BINOP_EQV) as an alternation ARM fails

### Symptom
After Fix 2, `while cur === A | cur === B do` still loops 0×. Narrowed precisely:
`if a === 5 | a === 6 then` → SCRIP "N", oracle "Y". `==` and `=` in the identical alt position
WORK; single `===` works; a 3-arm `a===5|a===6|a===5` works; the 2-arm case fails.

### Why it matters for self-host
`parse.icn`'s `parse_locals` is `while parse_tok_rec === lex_LOCAL | parse_tok_rec === lex_STATIC
do`. So ANY Icon program with a `local`/`static` declaration currently fails to compile through
SCRIP-jtran with "Expecting ;/end". hello_t (no locals) is why the base case passes.

### Diagnosis so far
`--dump-ir-verbose` on `if a === 5 | a === 6` shows a MALFORMED operand graph for the two
`BINOP_TEST binop=22` nodes vs the clean `binop=9` (`==`) case: the `==` version lowers each arm as
a self-contained `VAR;LIT;BINOP_TEST(ω→dj)`; the `===` version cross-wires the two tests into each
other and into the inner DISJUNCTION node (test operands reference the disjunction itself). Lowering
of the two operators is IDENTICAL at the `is_relop` binop site (line 298-314) and the runtime
`rt_jct_relop_impl` EQV branch is correct (single `===` works) — so the fault is in how binop +
alternation COMPOSE for EQV specifically, in the operand/edge construction, NOT the comparison.
Tightening the `opfail` ω-override (line 308) to generator-only had NO effect (reverted).

### Next step
Untangle the `lower_alt` × binop operand graph for the 2-arm EQV case: find why the left `===`
arm's `BINOP_TEST` operands/edges get scrambled (the IR shows test-references-disjunction). Compare
node-by-node against the `==` case. This is a real lowering bug in binop/alternation composition,
distinct from Fixes 1 & 2. Fixing it should immediately unblock `local`/`static` parsing and let
the full self-host proof extend to non-trivial programs (set ops, locals, loops).
