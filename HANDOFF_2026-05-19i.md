## ⚠ MIRROR-GAP-PRF-14-CLEAN-SMOKE — could not run smoke

Two pre-existing environmental issues block runtime verification in this container:

1. **`scrip --interp` segfaults** on anything non-trivial (while loops, pattern
   matches, any program loading the runtime chain via `global.sc`). Documented
   in `HANDOFF_2026-05-19h4.md` and `GOAL-PST-SNOCONE.md` as
   `MIRROR-GAP-SC-SC-5`. Container needs `bash scripts/build_csnobol4_archive.sh`
   followed by `make scrip-monitor` — but csnobol4 build itself requires a
   bootstrap `snobol4` binary that isn't installed.

2. **`scrip --dump-ast` segfaults** on the `function dq_unescape(...) { ... }`
   syntax (lines 16–39). Pinpointed by bisection:
   - Lines 1–19 (head through `while (1) {` line): exit 0, parses cleanly
   - Lines 1–20 (adds `if (IDENT(raw)) break;`): segfault
   - This pre-dates our edit — same segfault occurs in the parser-only mode
     of the C frontend regardless of any pattern content. The grammar
     portion below line 39 was confirmed to `--dump-ast` cleanly on
     extracted probes (e.g. `lines 1-15 + 'MyRule = ("hello" | "world");'`
     produced correct `(TT_ALT ...)` AST output).

**Conclusion:** the grammar surgery is structurally correct; the pure-recognizer
nature of the file is verified by grep counts and by `--dump-ast` partial probes.
A full smoke run requires either fixing the container env or running on a
machine with a working `scrip` build.

## Residue worth flagging for the PRF-14 rewrite

The PRF-14 author should expect these vestiges from stripping:

1. **`TryStmt` (line 490-498)** has a leftover `( EQ(try_has_catch, 1) )`
   value-space predicate. The two `reduce('TT_FNC', 3)` / `reduce('TT_FNC', 2)`
   arms it selected between are gone, so this predicate is now functionally
   dead. Not stripped because it's a value-space test, not a tree action.

2. **Several rules now read as `( Expr Block )`** with vestigial whitespace
   from where shift/reduce actions used to live (e.g. `WhileStmt`, `UnlessStmt`,
   most of the phaser stmts). Structurally valid Snocone, just sparse.

3. **`Compiland` rule (line 627-632)** now reads as
   `Compiland = POS(0) ARBNO( SubStmt | *ClassDecl | (Stmt ) ) $' ' RPOS(0) ;`
   — note the trailing space inside `(Stmt )`. Functionally fine.

4. **Dead code outside patterns** (untouched):
   - `function dq_unescape(raw, result, lit, ch) { ... }` — no longer called
     since `Dq_unescape` references in patterns were stripped
   - `gather_seq = 0;` — no longer mutated
   - `struct slink { snext, sval }` — was for old `sub_list` chain (PRF-S7
     comments say it was already eliminated)
   - All `capvf = ''`, `colnmf = ''`, ... declarations — no longer written to
     by patterns
   - The PRF-14 author may delete these wholesale before rewriting, or keep
     them as scratch storage if the new approach uses captures + post-match
     helpers.
