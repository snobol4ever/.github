## Failed experiments dropped

Three iterations of an auto-fixup script (`ec_uni_8_3_fixup{,_v2,_v3}.py`)
were tried during EC-UNI-8.3 to mechanically promote one-liner pat fns.
All three had bugs:
- v1: `" ".join(text.split())` collapsed whitespace inside string literals
  (would silently change WASM/Jasmin output)
- v2: single-fn-only; couldn't handle multi-fn files
- v3: lost inter-arm local variable declarations (e.g.
  `const char * s = ...; int kind = ...;` between IS_X86_BIN and IS_JVM)

All three were deleted; the 4 sm_pat_* files were hand-promoted (af370d45).
If a future opcode addition needs the promoter, write a fourth version that
preserves both whitespace and inter-arm preludes.
