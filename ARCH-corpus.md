## What Lives Here

```
programs/
  snocone/          Snocone reference: snocone.sc, snocone.snobol4, report.md
  rebus/            Rebus test files: word_count.reb, binary_trees.reb, syntax_exercise.reb
                    ← ADD: oracle .sno outputs for round-trip test (Step R12)
  gimpel/           Gimpel library programs
  shafto/           Shafto AI corpus
  beauty/           beauty.sno + all 19 -INCLUDE helper libraries
  oracle/           Cross-engine oracle suite

crosscheck/
  rung2/            210_indirect_ref, 211_indirect_assign, 212_indirect_array
  rung3/            310_concat_strings, 311_concat_numeric, 312_concat_null
  rung4/            410_arith_int, 411_arith_unary, 412_arith_real,
                    413_arith_mixed, 414_remdr
  rung8/            810_replace, 811_size, 812_dupl
  rung9/            910_convert, 911_datatype, 912_num_pred,
                    913_integer_pred, 914_lgt
  rung10/           1010_func_recursion, 1011_func_redefine, 1012_func_locals,
                    1013_func_nreturn, 1014_func_freturn, 1015_opsyn,
                    1016_eval, 1017_arg_local, 1018_apply
  rung11/           1110_array_1d, 1111_array_default, 1112_array_multi,
                    1113_table, 1114_item, 1115_data_basic, 1116_data_overlap
  beauty/           101_comment … 140_self  (rung 12)
```

Each crosscheck file is a self-contained `.sno` + matching `.ref` oracle output  
(generated by CSNOBOL4 2.3.3). A PASS line is the only stdout on success;  
any FAIL line names the exact assertion. Numbers leave room for insertion.

## How Each Repo Uses This

- **snobol4jvm:** submodule at `corpus/lon`
- **snobol4dotnet:** submodule at `corpus`
- **snobol4x:** referenced via `$INC` path for `-INCLUDE` files

## Update Protocol

Never add per-platform files. If a `.sno` file is used by more than one platform,  
it goes here — not in the platform repo. When adding oracle outputs for Rebus  
(Step R12), add them here. Crosscheck `.ref` files are always generated by  
`snobol4 file.sno > file.ref` under CSNOBOL4 2.3.3 — never hand-written.
