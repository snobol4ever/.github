# HANDOFF 2026-05-20 — ST2-1: stage2_t baton, g_stage2 global value

**Session:** GOAL-HEADQUARTERS, ST2 rung created and ST2-1 completed.
**Developers:** Lon Jones Cherryholmes · Claude Sonnet 4.7.

## What landed

`871d2f0b` (one4all): ST2-1 — Stage 2 handoff (lower → interp/emit) is now
a single named struct `stage2_t`, materialized as the global value `g_stage2`
(in `.bss`).  `SM_sequence_t` restored to its original sequence-only shape:
the SM opcode array and what the array needs.  `stage2_t` embeds the
`SM_sequence_t` by value and adds the four previously-sidecar tables
(`label_table`, `proc_table`, `pl_pred_table`, `module_registry`) plus the
program-level `lang`.

`lower()` reads top-to-bottom as what we wanted:

```c
stage2_t *lower(const tree_t *prog) {
    if (!prog || prog->t != TT_PROGRAM) return NULL;
    stage2_reset();
    g_p = &g_stage2.sm;
    polyglot_init(&g_stage2, prog, polyglot_lang_mask(prog));
    ... main lowering pass writes into &g_stage2.sm via g_p ...
    g_stage2.lang = g_lang;
    return &g_stage2;
}
```

Six reader shim macros (`label_table`, `proc_table`, `g_pl_pred_table`,
`g_registry`, `label_count`, `proc_count`) redirect legacy names into
`g_stage2` so existing readers compile unchanged.  Each macro annotated
`ST2-1 reader shim: deleted in ST2-1b`.

`g_current_SM_seq` and `g_current_stage2` both dissolved — there is one
struct (`g_stage2`), one address-of expression, no synchronization.

## Why three honest pivots happened mid-session

Recording the trail so it isn't repeated:

1. **First attempt:** add `s2_*` fields to `SM_sequence_t` and populate them at
   the *end* of `lower()` (snapshot the existing globals).  Rejected by Lon as
   kludgy — the populate-at-end didn't communicate "we are building this
   thing", it just labeled the result.
2. **Second attempt:** `typedef stage2_t = SM_sequence_t` alias.  Required
   keeping `g_current_SM_seq` and `g_current_stage2` in lockstep — two names
   for the same thing.  Lon rejected the alias-pair synchronization.
3. **Third attempt (landed):** two structs, two purposes.  `SM_sequence_t` is
   the array, `stage2_t` is the baton, `g_stage2` is a global value.

The SR-15c lesson (LowerCtx rollback) was respected throughout: lower's
*internal* pass state stays file-scope `static` in `lower.c`.  ST2 addresses
the *output*-side handoff only — the inverse of LowerCtx.

## Honest scope notes for next session

Three follow-ons deliberately deferred to keep this commit small and
auditable:

- **Fixed-size inline arrays.** `label_table[STAGE2_LABEL_MAX]` (4096) and
  `proc_table[STAGE2_PROC_TABLE_MAX]` (256) are hard caps and ~150KB of
  `.bss` for hello.sno.  Lon flagged this at handoff time — "Why limit the
  arrays inside the structs?  Does it not simply limit the space?"  Right
  answer: convert to dynamically-grown arrays (same `_grow` pattern as
  `SM_sequence_t.instrs`).  Tracked as **ST2-1c**.
- **Six reader shims still in place.** Migrating readers to take
  `stage2_t *s2` directly and deleting each macro is **ST2-1b**.  The macros
  are temporary scaffolding, not a permanent fixture.
- **No link-time isolation gate.** Header firewalls don't prove sidecar
  isolation any more than ISO-2/3 prove parser isolation.  **ST2-2**
  parallels ISO-7.
