## M2 ORACLE RE-BASELINE (the rung's second half — src/interp/IR_interp.c)
Pre-existing m2 bug, the cut analog of the documented ITE-commit oracle bug: redo through a cut RE-SUCCEEDED (probe `f(X):-X=a,!. f(X):-X=b. main:-f(Q),write(Q),nl,fail;true.` printed infinite `a`). Mechanism, fully traced: (1) the cut's forward case ran `resolve_cp_truncate(g_resolve_cut_barrier)` on EVERY execution — and `IR_interp_resume` re-walks bodies from the ENTRY with det nodes as idempotent passthroughs, so each redo re-truncated, FREEING the enclosing choice's own CP and any post-cut CPs (resolve_cp_truncate frees nodes); (2) IR_CHOICE's `spine_says_live` read `resolve_cp_current() != zc->cp` — true precisely BECAUSE the cut truncated zc->cp away, so the COMMIT MARKER was misread as inner liveness; (3) resume then re-walked `[unify, cut]`, every node passed through, the body re-succeeded vacuously with stale bindings. Fix, three parts, all in IR_interp.c:
- A. IR_CUT is stateful: state==0 → truncate + flag + state=1 + γ; state>0 → γ only (idempotent passthrough; protects post-cut CPs from re-truncation). bb_reset clears state per fresh activation; snapshot/restore preserves it across the caller's save/restore dance.
- B. `bb_body_live_choice_cut_aware(bbg)` (new, next to bb_body_has_live_choice): if the body's GCONJ has a passed cut (state>0, LAST such index), only the goal-suffix AFTER it can offer alternatives; else defers to the old scan — byte-identical behavior for cut-free bodies.
- C. IR_CHOICE redo: `cp_cut_away` = zc->cp set but no longer on the CP stack (walk from resolve_cp_current() via ->parent, matching pointer AND ->resume==(void*)bb to defeat malloc reuse) → COMMITTED: gates spine_says_live, commit-fails instead of clause-advancing, and resume-failure commits on `flag || cp_cut_away`. IR_GOAL's redo guard uses the cut-aware scan (covers single-clause cut-last: firstpath under fail-driven redo).
Evidence the re-baseline is sound: GATE-3 exactly 115/115 m2 · 18/0/97 m3 · 105/0/10 m4 and GATE-1 honest 136/0/0 — IDENTICAL to the pre-fix baseline at aa587c9; six-shape probe battery all ISO-correct (multi-clause commit `a`; single-shot `a`; no-cut sanity `a b`; redo-past-cut `a b`; cut-not-reached clause-2; firstpath-redo `b`).

## LESSONS
- The cut is genuinely ZERO new semantics on the GZ path — one ternary in the emitter plus a det-shaped box. Everything hard lived in the m2 oracle.
- When a CP-stack and a node-state layer coexist, a commit expressed in one (truncated CP) reads as its OPPOSITE in the other (liveness). Detect commitment explicitly (cp_cut_away), never by inequality side-effects.
- resolve_cp_truncate FREES — never deref a stored resolve_choice* after a cut may have run; pointer+owner match on the live stack only.
- The 5c lesson held: no bb_prepare needed when a box reads no operands; op_ival stomping is only a hazard for nodes carrying payloads.
