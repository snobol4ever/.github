## ⛔ THE CONVERSIONS (CV1–CV10) — apply every applicable one until `audit_bb_fixup_file.sh <file>` rc=0
A CONVERSION = a spec of what to TARGET and what to CHANGE it to. (R1–R6 are aliases CV1–CV6.)
- **CV1 — TERSE COMMENT.** Each `x86("comment", …)` → the bare `IR_<KIND>` the box dispatches from. Terseness only.
- **CV2 — INLINE MULTI-EMIT HELPERS.** Every helper composing ≥2 `x86`/`x86_*` emission calls (not the `bb_*` entry) inlined at its call sites + deleted. KEEP zero-emit fragment builders (label/value/slot computers spliced INTO one `x86()` operand) and the `x86()` primitive.
- **CV3 — DE-ESCAPE ins\*.** `x86("ins1"|"ins2"|"ins3"|"Lins1"|"Lins2", …)` are TEXT-only escape hatches (zero bytes in BINARY = ONE-MEDIUM violation) → replace each with a real byte-emitting `x86()` form. Cat A (existing encoder) trivial; Cat B needs a new XK_RIPLBL/XK_LBLDIFF encoder added + byte-verified in x86_asm.h first.
- **CV4 — V2 HYGIENE.** ONE return per PLATFORM · kill signature-line decls + static-helper constellations · drop MEDIUM_* head wrappers · 1-src-line == 1-asm-line · ≤200 col · purge non-separator C comments (KEEP only the CV1 box-name comment) · zero blank lines · add missing 200-char separators.
- **CV5 — ABSOLUTES.** zero raw bytes / MEDIUM_BINARY anywhere · zero emit_fmt() · real Greek α β γ ω · EMIT-BLIND (no neighbor inquiry). A file that can't meet this without losing logic is a ONE-IR-ONE-LOGIC violator → execute the split IN PLACE (per SOP; there is NO defer/pin-and-wait).
- **CV6 — NO LOCALS IN TOP-LEVEL TEMPLATE FUNCTIONS.** ZERO local-variable decls in the template function body. Values arrive via zero-emit value-computer helpers (helpers MAY have locals/params), `_.op_*`/`_.bb_*` slots, or **lambda parameters (parameters are not locals)**.
- **CV7 — VALID x86/x64 ASSEMBLY ONLY** (Lon 39th run, verbatim "only VALID x86/x64 assembly instructions are ALLOWED as 1st parameter of x86 function"). TARGET: any `x86("XXX", …)` where XXX is not a real mnemonic (mov/lea/add/cmp/jmp/je/call/ret/…) or a directive spelled as in the .s (.quad/.string). CHANGE: real spelling + an operand form mapping to the same encoder; decompose multi-line pseudo-ops per-line; encoders + medium switch stay inside x86_asm.h. Precedents: `ro_load_q`→`x86("mov", reg, ROQ(n))` (XK_ROSLOT); RO seal decomposes to `x86("def", L(n))+x86(".quad", …)` (quad) / `+ x86(".quad", LS(n), lit)+x86("label", LS(n))+x86(".string", lit)` (string) — byte-identical by construction. The `x86_ro_seal_*` free fns stay inside x86_asm.h.
- **CV8 — PLATFORM IF STATEMENTS PRESERVED.** Each `if (PLATFORM_*)` is its OWN separate if statement; PLATFORM_X86 stays fenced from JVM/.NET/JS/WASM (which stub out but keep the fence shape). "ONE return per PLATFORM" = one return INSIDE each platform's if (+ the empty fallthrough return), NEVER one merged return spanning platforms. The platform predicate never appears inside an IF() combinator arg.
- **CV9 — NO PARAMETERS / NO `_str` ON TEMPLATE FUNCTIONS.** Conformant shape = `std::string bb_X()` — NO `_str` postfix, ZERO parameters, NO extern-C wrapper (the `bb_alt` precedent: declared `std::string bb_alt();` in bb_templates.h, invoked `bb_emit_x86(bb_alt())` in emit_core). Rename `bb_*_str`→`bb_*`, drop all params. The audit checks CV9 (`bb_*_str` / `IR_t*` in sig).
- **CV10 — NO IR-GRAPH ACCESS FROM TEMPLATES.** No `pBB`, `_.node`, `g_emit_cfg`, `bb_operand_aux_get`, `IR_t` deref, `IR_LIT(node)`, child/operand walks — top level OR helpers. By call time EVERY value the template needs is in `g_emit` (scalar or collection); the driver-side `bb_prepare` (emit_bb.c) does ALL graph inspection and delivers via `_.op_*`/`_.bb_*` (add sm_emit_t fields as needed; precedent op_parts_n/tag/ival/str for IR_ALT). Zero `IR.h` surface in a template = the completion sign. The audit checks CV10 (`ir_call_arg`/`IR_LIT`). **Exception:** an arbitrary-depth term-tree builder that can't be pre-flattened (e.g. `emit_build_compound_term`, 15 callers) lives in a CV10-exempt support TU (`src/emitter/emit_term_build.cpp`), NOT in a bb_*.cpp.

## ⛔ IDIOMS & FINDINGS (durable across the lap)

So `emit_intern_str(cs)` can be dropped and `strtab_label` made the sole label source. Re-verify each session (it's the one idempotency hazard).
- **The notany hazard:** when a string-cset template calls `strtab_label`/intern UNCONDITIONALLY before a dispatch branch, the intern side-effect (a `.S<n>` slot) fires even on branches that never use the label. Folding `strtab_label` into a branch-only path renumbers all later `.S` labels → m4 divergence. Keep `strtab_label` as an unconditional bare statement (or a comma-operator positioned before the branch). Caught by A/B, not by construction.
- **Shape idioms** (all proven, byte-identical):
  - single-linear string-cset, bare `strtab_label`: bb_match_break / breakx / notany / any.
  - dispatch + comma-operator `strtab_label` (intern before a branch): bb_match_capture; empty-guard-as-ternary (silent `std::string()`, not a bomb): bb_match_atp.
  - `std::string` label arithmetic with no locals → single capturing lambda `[&](std::string base){ return …; }(<local-free expr>)`: bb_match_arbno. (CV6: lambda params are not locals; the audit excludes capturing-lambda returns from the returns count, so a capturing lambda beats a static helper-function which would ADD a counted return.)
  - scratch-slot `FR(_.x86_scratch_off + N)` inlines: bb_match_arb.
  - fn-ptr call (drop `@PLT`): `x86("call", "fn", (uint64_t)(uintptr_t)(void *)(<sig>)fn)`. If a scalar-arm rt_* isn't declared in bb_common.h, add the extern.
