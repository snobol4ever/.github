================================================================================
CRITICAL ARCHITECTURAL INSIGHT
================================================================================

**Problem:** Initial design assumed scalars would emit via IR_t (like patterns)

**Discovery:** Lower phase emits SM_Program (stack machine instructions)
  directly for scalars. IR_t is only for patterns (generators).

**Solution:** Walk SM_Program instruction stream instead of IR_block_t
  - emit_js_from_sm() maps SM_* opcodes to JavaScript
  - Prologue/epilogue handle switch/dispatch structure
  - Statement numbers tracked from SM_STNO instructions
  - Pattern factories (IR-based) + scalars (SM-based) coexist cleanly

**Validation:** 6/6 smoke tests pass with this architecture.
