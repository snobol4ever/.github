## Design decisions made this session

**TT_COMPOUND for compound terms:** functor shifted as `TT_FNC(name)` leaf, then
`reduce('TT_COMPOUND', nTop())` pops functor + all args. Lower converts to
C-parser's `TT_FNC(name, [args])` shape. This avoids needing `reduce_call` or
`ReduceCall`.

**TT_MAKELIST for lists:** nil = `TT_MAKELIST(0 children)`, `[h1..hn | tail]` =
`TT_MAKELIST(n+1 children: elems then tail)`. Matches C parser's `pt_list` shape.

**TT_BINOP for generic ops:** ops captured via `_op_name` token alias produce
`TT_BINOP` with empty v — lower maps to correct op using context. Dedicated kinds
used for all statically known operators.

**DCG rules:** SCRIP parser emits raw `TT_DCG_RULE(head, body)`. Lower (prolog_lower.c,
already clean from Phase 1) handles full DCG expansion. The 180-LOC DCG expansion
block (`dcg_fresh_var`, `expand_dcg_body`, etc.) is fully deleted.

**ascii_table kept:** The char-code lookup `"0'c"` → `TT_ILIT(ascii_table[c])` still
needs the ascii_table init block. The table is pure data, not a tree-building side effect.
