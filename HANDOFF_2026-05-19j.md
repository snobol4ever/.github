## ⚡ Next step — PRF-14 step 2: attach tree actions

The grammar foundation is done. Step 2 adds **two patterns** to every rule:

1. **`shift(token_pattern, "'TT_KIND'")`** — match the pattern, push a leaf of
   the named kind onto the tree stack. Used for terminals (idents, literals,
   variables).
2. **`reduce("'TT_KIND'", N)`** — pop N items from the tree stack, wrap them
   as children of a fresh node of the named kind, push back. Used after each
   alternative that should produce an interior node.
3. Plus the variable-arity supporting cast: **`nPush()` / `nInc()` / `nTop()` / `nPop()`**
   for variable-arity reduces (the n-ary collection over `ARBNO` and over
   right-recursive chains).
4. And **`assign(.var, val)`** for parser-scratch variables when a name needs
   to be captured and used in a later `shift`.

### What each kind maps to — see `PRF-14-TREE-SHAPE-ORACLE.md`

The oracle document lists every TT_* kind raku.y produces, with the exact
shape (arity and child order). Three categories:

- **38 shared kinds** (TT_ADD, TT_MUL, TT_EQ, TT_LT, TT_TO, TT_ASSIGN,
  TT_FIELD, etc.) — translate directly from existing expression-precedence
  rules.
- **28 dedicated raku.y kinds** (TT_SAY, TT_PRINT, TT_TRY, TT_UNLESS,
  TT_MAP, TT_GREP, TT_SORT, TT_DIE, TT_GATHER, TT_NEW, TT_METHCALL,
  TT_HASH_GET/SET/EXISTS/DELETE, TT_ARR_GET/SET, TT_CAPTURE,
  TT_NAMED_CAPTURE, TT_SMATCH, TT_TWIGIL_FIELD, TT_FOR_RANGE,
  TT_STMT, TT_SUB_DECL, TT_CLASS_DECL, TT_PRINT_FH, TT_SAY_FH,
  TT_DECL, TT_PRINT) — explicit mapping required per the oracle.
- **3 kinds the old SC parser had but raku.y doesn't** (TT_FOR, TT_RECORD,
  TT_RXLIT) — DO NOT introduce; raku.y is the oracle.

### Suggested ordering for step 2

1. Add `shift(...)` to each token/leaf reference (terminal-equivalent
   productions: Ident, VarScalar, LitInt, etc.) — they all need to push
   leaves of the appropriate kind.
2. Add `reduce(...)` to each rule alternative that produces an interior node.
   Start with the simplest: `if_stmt`, `while_stmt`, `until_stmt`, `repeat_stmt`.
3. For variable-arity rules (stmt_list, class_body_list, when_list,
   named_arg_list, param_list, arg_list, method_postfix chain),
   wrap with `nPush() ... ARBNO( nInc() *item ) ... reduce("'TT_X'", "nTop()") nPop()`.
4. For binary-op right-recursive chains, use `reduce("'TT_OP'", 2)` directly
   in each FENCE arm.
5. For `expr`, `cmp_expr`, `add_expr`, `mul_expr`: straightforward
   `reduce("'TT_OP'", 2)` per arm.
6. Re-verify grep counts, then attempt smoke when the env permits.
