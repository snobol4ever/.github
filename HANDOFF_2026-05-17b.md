## Three constructs landed

### IJ-ALT-BETA-SINGLESHOT (`66dc3582`)
**Root cause of rung18 infinite hang confirmed and fixed.**
IR_ALT β-path called `IR_exec_node(child[ci])` unconditionally. Literal nodes (IR_LIT_F, IR_LIT_I, IR_LIT_S) always succeed — they have no generator state and return their value every time. This trapped IR_BINOP_GEN's relop-retry `for(;;)` loop: relop fails → advance RHS (IR_ALT β) → IR_ALT resumes same literal → same value forever.

Fix: added `ALT_IS_GEN()` macro (mirrors IR_IS_GEN_KIND). IR_ALT β now only calls IR_exec_node on generator-kind children; single-shot children are treated as exhausted immediately and control falls through to next-child scan.

**rung18: 0/5 → 5/5** (including real_relop_goal hang fixed).

### IJ-TOBY-REAL (split across `66dc3582` + `6f3e9690`)
`TT_TO_BY` had no lowering case (NO-AST stub). IR_TO_BY executor was integer-only. Added:
- `TT_TO_BY` lowering in `lower_icn.c`: emits `IR_TO_BY` with c[0]=lo, c[1]=hi, c[2]=step

Semantics verified against JCON (`vBigInt.ToBy` — real-typed, uses `NLessEq`) and Icon classic (`omisc.r` — integer-only via `cnv:C_integer`). Our implementation follows JCON semantics.

**rung19 to-by: 0/2 → 2/2**.
