## Three constructs landed this session (RULES.md ceiling = 3)

### SCT-1b — Structured-control + label-sanitize (`e9ee90e4`)

`lower_sno.c::emit_stmt` now dispatches on `:subj` tag for TT_IF, TT_WHILE,
TT_DO_WHILE, TT_FOR, TT_RETURN, TT_PROC_FAIL, TT_NRETURN, TT_LOOP_BREAK,
TT_LOOP_NEXT. Lowering follows ARCH-SNOCONE.md §"Lowering map" precisely.

- **TT_IF** allocates `_Lelse_NNNN`/`_Lendif_NNNN` via in-emitter `if_seq`
  counter (C frontend PST mode keeps TT_IF as a tree — no pre-alloc).
- **TT_WHILE/TT_DO_WHILE/TT_FOR** read pre-allocated `_Ltop_NNNN`/
  `_Lcont_NNNN`/`_Lend_NNNN` from trailing TT_QLIT children stashed by
  the frontend.
- **TT_LOOP_BREAK/TT_LOOP_NEXT** read top of 64-deep loop-label stack.
- **TT_RETURN/TT_PROC_FAIL/TT_NRETURN** emit `:(RETURN)/:(FRETURN)/:(NRETURN)`.

Fixed two pre-existing bugs surfaced during this work: `:lbl` attr's
payload lives in `c[0]->v.sval` not the attr's own `v.sval`; `TT_GOTO_U`'s
target label same shape. Both fixed via new `label_of()` helper.

Added `label_sanitize()` — strips leading underscore per SPITBOL Manual
Ch.14 line 9335 ("Labels must begin with a letter or digit"). The Snocone
PST mode pre-allocates names like `_Ltop_NNNN` which SPITBOL rejects with
ERROR 230 and SCRIP `--sm-run` silently mangles. Threaded through every
label emission site via a 4-deep buffer ring so multiple sanitize() calls
in one printf format don't clobber.

Padded label-only statements with `OUTPUT =` so SCRIP's SNOBOL4 parser
(rejects label-only lines) accepts them.

**Result:** all 6 parser_*.sc → 0 `?TT_` placeholders.

### SCT-1c — SNOBOL4 line-continuation for >1024-char emissions (`c1074e5f`)

SPITBOL hard-fails with ERROR 226 on lines >1024 chars (Manual Ch.14 line
9022). `parser_snobol4.sc`'s `Expr17` is one 1908-char TT_ALT;
`parser_raku.sc` was 2233; `parser_icon.sc` was 1323.

Redirected `emit()` through 16KB in-context line buffer. `emit_nl()` scans
backward from offset 900 for a space/tab outside quoted strings
(`sno_in_quoted` walks from start because SNOBOL4 has no escape character
— every quote is structural). Splits at safe positions, separates chunks
with `\n+` per SPITBOL Manual Ch.14 line 9456+.

Safe-split reliability comes from `emit_expr` parenthesising every binary
op — every space between tokens IS a safe split point (precedence
preserved by the parens). 900 leaves headroom under the 1024 cap.

**Result on all 6 parsers:** longest physical line ≤899 chars (was up to 2233).

### SCT-1d — Multi-file `--dump-sno` + `-CASE 0` + dedup tail :(RETURN) (`b31e72e4`+`f097651a`+corpus `c97eeb0`)

Three orthogonal sub-changes that together make
`scrip --dump-sno <runtime files> parser_X.sc` produce a self-contained `.sno`:

1. **Multi-file mode** (`src/driver/scrip.c`): `--dump-sno` previously
   emitted the first file's AST and returned. Restructured to mirror
   `--dump-sm`: accumulate via `MERGE_AST` across all argv files, emit at
   end of multi-file loop. Applies to both the snocone/rebus/icon/raku/
   prolog branch AND the SNOBOL4 branch.

2. **`-CASE 0` prelude** (`src/lower/lower_sno.c`): SPITBOL Manual Ch.14
   line 9067+: case-fold-to-upper is default. Without `-CASE 0`,
   `ShiftReduce.sc`'s `Shift(t,v,s)` collapses with `semantic.sc`'s
   `shift(p,t)` → ERROR 217 duplicate label. RULES.md mandates byte-for-
   byte case sensitivity. SCRIP's SNOBOL4 frontend is already
   case-sensitive (no-op control).

3. **Suppress redundant tail :(RETURN)** (`src/lower/lower_sno.c`): when a
   function body ends with explicit `return;` we emit `:(RETURN)`, then
   the Gimpel template tail emits ANOTHER. `emit_nl` sets
   `last_was_return` when the buffered line ends with
   `:(RETURN)/:(FRETURN)/:(NRETURN)`; template tail skips when set.
   Saved 56 redundant lines on `full_snobol4.sno` (941→885), 27% of
   `:(RETURN)` lines eliminated.

4. **Wrapper** (`scripts/run_parser_sync_monitor.sh`): now passes
   `global.sc tree.sc stack.sc counter.sc ShiftReduce.sc semantic.sc
   tdump.sc` alongside the parser. Added 1024-char line-length sanity warning.

5. **Corpus rename** (`corpus/SCRIP/semantic.sc`): `_qtag` → `qtag`.
   Only underscore-leading function name in the seven runtime `.sc` files.
   SPITBOL rejects underscore-leading identifiers in `DEFINE()` with ERROR 230.

**Result:** `scrip --dump-sno <runtime> parser_snobol4.sc` produces 885
lines that **SPITBOL parses cleanly end-to-end** (zero parse errors).
Single runtime `ERROR 041` at line 873 (`n(ptree)` on non-tree value
— downstream parsing-execution issue, not a transpile defect).

3. **The `:(RETURN)` dedup** has a subtle edge case: it triggers per
   emit_nl based on the JUST-FLUSHED line. If a TT_DEFINE block contains
   another TT_DEFINE inside (nested function), the inner one's
   `_end OUTPUT =` last-was-return state gets passed through to the
   outer's tail check. Currently no Snocone code nests functions (and
   the grammar may not even permit it), so this is theoretical, but
   worth a note.

4. **`run_parser_sync_monitor.sh` not yet end-to-end driven** — it
   builds the .sno correctly but the downstream `test_monitor_2way_
   spitbol_vs_sm.sh` requires the SPITBOL IPC patch and the SCRIP
   monitor IPC libraries built. SCT-1f.

5. **The grammar-level constraint** ("only six runtime primitives in
   parsers: shift, reduce, nPush, nInc, nTop, nPop; no helpers_*.sc")
   is about the .sc INPUT shape — the emitter is unconstrained. SCT-4
   (icon) and SCT-5 (raku) still need their helpers files deleted FIRST,
   which means rewriting the parsers to inline those helpers.
