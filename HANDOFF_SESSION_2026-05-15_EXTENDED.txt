BLOCKER IDENTIFIED: SM_PAT_CAPTURE_FN_ARGS Opcode Mismatch
================================================================================

THE MYSTERY:
- SM dump displays: "SM_SUSPEND_VALUE s="IDENT" nargs=2"
- Actual opcode value at runtime: 66
- Enum maps opcode 66 to: SM_PAT_CAPTURE_FN_ARGS

ROOT CAUSE:
Lower layer emits SM_PAT_CAPTURE_FN_ARGS for builtin function calls
(IDENT, DIFFER, SIZE, TABLE, DEFINE, etc.), not SM_SUSPEND_VALUE.
SM dump's string names use different convention than enum values.

IMPACT:
- Builtin function calls missing rt.set_last_ok(...) line
- Success/failure not propagated for builtins
- Keywords tests fail (IDENT, DATATYPE, ALPHABET, etc.)
- Blocks progress to 120+ PASS

INVESTIGATION DONE:
1. Added case SM_PAT_CAPTURE_FN_ARGS to switch statement
2. Added debug output to track case matching
3. Debug output never fired - case doesn't match at runtime
4. Hypothesis: enum ordering or compilation mismatch

                      instr->op, SM_PAT_CAPTURE_FN_ARGS);
